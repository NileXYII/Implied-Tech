Why KENAI

Everything the M3 spec defines.
Nothing extra.

Built token-first from the Material Design 3 guidelines. Every color role, every easing curve, every component shape — exactly as Google specified.

palette
Full M3 Color System

30+ color roles — primary, secondary, tertiary, error, and the full surface container hierarchy. Light and dark themes with tonal elevation overlays.

text_fields
15-Role Typography Scale

Display, Headline, Title, Body, Label — 3 sizes each with exact M3 optical sizes, weight, tracking, and line heights. Google Sans + Roboto.

animation
M3 Motion Tokens

16 duration steps and 9 named easing curves — Standard, Emphasized Decelerate, and Spring — wired as CSS custom properties.

water_drop
Ink Ripple Engine

Pixel-perfect M3 ripple via kenai.js. Add .ripple to any element. State layers (8% hover, 12% pressed) are pure CSS.

dark_mode
Automatic Dark Mode

One data-theme attribute. Respects prefers-color-scheme, persists to localStorage, fires a custom event.

bolt
Zero Build Step

Pure CSS custom properties and vanilla JS. No Sass, no PostCSS, no webpack. Two files, link them in, done. Override any token with one line.

Add data-theme="dark" to your <html> element. The JS runtime auto-detects prefers-color-scheme, persists the user's choice to localStorage, and fires a kenai:themechange event.
